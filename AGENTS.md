# Repository instructions

## Changelog

- Every change under `lib/` must update the `## [Unreleased]` section in
  `CHANGELOG.md` in the same branch.
- Create the `Unreleased` section immediately above the latest release if it
  does not exist.
- Use the appropriate Keep a Changelog category and describe the effect on
  users rather than restating commit messages or implementation details.
- Do not edit a released section. Release preparation is the exception: rename
  `Unreleased` to the new version and date after confirming its entries.
